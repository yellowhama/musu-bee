## Wave 2: musu-core 안정성

### W2-1: checkout_issue 원자화 (TOCTOU 수정)
**파일:** `musu-core/src/musu_core/backends/local.py`

현재:
```python
existing = self.get_issue(issue_id)  # read
if existing is None: return None
self._db.execute("UPDATE issues SET ... WHERE id=?", ...)  # write (race window here)
return self.get_issue(issue_id)
```

수정 목표: UPDATE + 영향 행 수 확인으로 원자화:
```python
cursor = self._db.execute(
    "UPDATE issues SET checkout_by=?, checkout_at=..., status='in_progress', updated_at=... WHERE id=? AND checkout_by IS NULL",
    (agent_id, issue_id),
)
if cursor.rowcount == 0:
    return None  # not found OR already checked out
return self.get_issue(issue_id)
```

## Wave 3: musu-connects daemon 명령

### W3-1: `daemon` 서브커맨드 추가
**파일:** `musu-connects/apps/musu-connectsd/src/main.rs`

```
musu-connectsd daemon [--bind <ip:port>] [--bridge-url <url>] [--mdns] [--stun <server>]
```

동작:
1. QuicProvider 초기화 (QUIC endpoint bind)
2. SyncOrchestrator::new(quic_provider, SyncOrchestratorConfig::default())
3. `if --mdns`: MdnsService 시작 → 발견된 피어를 orchestrator.register_peer()에 등록
4. `tokio::spawn(orchestrator.run())` — heartbeat + route sync 루프
5. StunService로 외부 IP 탐색
6. 무한 대기 (ctrl+C 처리)

**Cargo.toml:** `tokio::signal` for shutdown

### W3-2: mDNS 발견 → SyncOrchestrator 연결
**파일:** `musu-connects-core/src/application/mdns_service.rs`

`MdnsService::discovered_peers()` → `orchestrator.register_peer()` 루프.
현재 MdnsService는 발견만 하고 orchestrator에 연결되지 않음.
