## F2 — HIGH: Supabase Management API PAT in git history

**Files committed**:
- `plans/REMOTE_MACHINE_RUNBOOK_2026-04-18.md` (commit `05e263ee`)
- `plans/PHASE9_P5_REMAINING_2026-04-18.md` (commit `6e03598a`)

**Token**: `sbp_17e09dee519f531792828842177d4cd43ca92507`
**Type**: Supabase Management API personal access token (`sbp_*`)

Supabase Management PATs can list/modify projects, query databases, and access
project settings. This token is now in git history and must be treated as
compromised regardless of whether the repo is public or private.

**Remediation (in order)**:
1. **Revoke now**: Supabase Dashboard → Account → Access Tokens → delete this token
2. Replace in both plan files with `<YOUR_SUPABASE_PAT>` placeholder
3. For the git history: since this is not a public repo and the token will be
   revoked, `git filter-repo` is optional. If the repo is or becomes public,
   purge the commits with `git filter-repo --path plans/ --invert-paths` is
   NOT right (would remove all plans). Instead use:
   ```bash
   git filter-repo --replace-text <(echo "sbp_17e09dee519f531792828842177d4cd43ca92507==>REDACTED_SUPABASE_PAT")
   ```
   Then force-push. Coordinate with any other clones.

## F3 — INFO: First-party CI actions not SHA-pinned

`actions/checkout@v4`, `actions/setup-node@v4`, `actions/github-script@v7` are
tag-pinned, not SHA-pinned.

First-party GitHub Actions (`actions/*`) are widely accepted at tag versions
since they are maintained by GitHub itself and tags are immutable in practice.
Third-party actions (`dtolnay/rust-toolchain`, `Swatinem/rust-cache`) are already
SHA-pinned. **No action required.**
