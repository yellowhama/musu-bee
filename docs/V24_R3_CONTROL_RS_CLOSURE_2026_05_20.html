§1 What R3 was

R3 replaces the standalone Python musu-control MCP stdio process (registered by Claude Code via .mcp.json and invoked as python -m musu_control) with a native Rust musu control subcommand on the single musu-rs binary. The new subcommand serves Anthropic's rmcp 1.7 stdio JSON-RPC server and exposes 13 tools (8 T1 native proxies + 5 T2 deprecated stubs) that talk to the R1 bridge via reqwest over loopback HTTP. Token resolution reuses the R6 install resolver via the new shared install/token.rs module — dedupes one of two pre-existing copies (R6 Auditor QB2 backlog) as part of R3.

Frame correction (load-bearing): musu-control is NOT a set of /api/control/* bridge routes. The original V24 master plan §5-R3 row implied this; Phase 0 Researcher F1 clarified the actual scope. R3 ports a STDIO PROCESS, not HTTP routes. Closes one of the R-cleanup outstanding gaps: Python musu-control can now be deleted at R10.
