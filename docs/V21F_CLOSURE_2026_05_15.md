## What shipped in 21.F

### Bridge HTTP surface — `musu-bridge/axis_routes.py`

Three read-only endpoints feeding the new UI views:

| Method | Path | Returns |
|--------|------|---------|
| GET | `/api/machines` | list of machines + capacity snapshot + in-flight queue count |
| GET | `/api/machines/{machine_id}` | single machine + active requests (bound/running) |
| GET | `/api/companies/{company_id}/dispatch` | agents grouped with their in-flight requests + totals |

All hit the v21.C `resource_requests` and v33 `machines` + v35
`machine_capacity` tables read-only. Auth reuses the bridge's
existing bearer-token middleware. Wired in `server.py` after the
v21.B watch router.

7 tests in `test_axis_routes.py` pin the contract: list + detail +
404 paths + company dispatch grouping + totals correctness.

### Next.js pages — `musu-bee/src/app/{c,m}/[id]/page.tsx`

**Company view (`/c/[id]`)** — agent list grouped by company, each
agent shows status, adapter type, and any in-flight resource_requests.
Links from a request's `bound_machine_id` jump to `/m/<id>`. Header
shows totals: agents_total / active / requests_pending / running.

**Machine view (`/m/[id]`)** — capacity bars (GPU VRAM, memory,
CPU idle %, runtime classes) + list of active requests on this
machine. Each request links back to `/c/<company_id>`.

Both pages:
- 5s polling safety-net `setInterval`
- SSE `EventSource` on `resource_requests` (and `machines` on the
  Machine view) for instant wake on state changes
- Loading / error / empty states match the dashboard convention

### Playwright spec — `musu-bee/e2e/v21-axis-views.spec.ts`

Stubs `axis_routes` via `page.route()` so the spec runs without a
live bridge. Validates:
- Company page renders agents + totals
- Cross-axis nav: company → machine via the bound link
- Machine page renders capacity bars + active requests
- Company link from machine request returns to `/c/[id]`
- 404 surfaces as the `role="alert"` error box

Spec compiles clean against the project tsconfig (`npx tsc --noEmit`
returns 0 errors).
