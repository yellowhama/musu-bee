## What shipped

### Schema v36 (Const III batch-approved)

`agents.isolation_profile TEXT` — nullable JSON column. `NULL` =
legacy / no sandbox. JSON `{}` = sandbox with empty allowlists (deny
everything). JSON populated = full `IsolationProfile`.

`_v36_up` idempotent via `PRAGMA table_info` gate; `_v36_down` tries
`ALTER TABLE … DROP COLUMN` (SQLite ≥ 3.35) and silently no-ops on
older builds. 7 tests pass.

### IsolationProfile (the data type)

```rust
pub struct IsolationProfile {
    pub allow_read:  Vec<PathBuf>,
    pub allow_write: Vec<PathBuf>,
    pub allow_net:   Vec<NetEndpoint>,
    pub cpu_secs:    Option<u32>,
    pub mem_mb:      Option<u32>,
    pub strip_env:   Vec<String>,
}
```

Default = deny-everything (all `Vec`s empty, no resource caps). JSON
serializable via serde; round-trips through SQLite TEXT column.

## What's deferred (still in-progress, per task list)

- **#296** Linux impl — `clone(CLONE_NEWUSER | CLONE_NEWNS | …)` +
  `/proc/self/uid_map` + bind-mount allow_read/write + seccomp filter
  + drop caps. Needs a Linux/WSL2 host to validate the pipeline
  doesn't ENOSYS on Ubuntu 22.04 / Fedora 39 / Alpine 3.19.
- **#297** Windows impl — `CreateAppContainerProfile` + ACL allowlist
  + Job Object with kill-on-close + restricted token. Validatable on

  permission testing.
- **#298** macOS impl — sbpl renderer + `sandbox-exec` wrap. Needs a
  macOS host for end-to-end validation.
- **#302** Per-platform isolation tests — land with each impl.
- **#303** Const VI Linux 3-distro bench — depends on #296.

## Why scaffold-first

The Rust trait and the v36 schema are the *contracts* the rest of the
system depends on. Once they're frozen, downstream work (21.E
`MachineReconciler` reading `agents.isolation_profile`, the v22 CEO
ramp-up of new agents with non-NULL profiles) can proceed in parallel
on whatever host we have. Platform impls iterate independently
without re-shaping the API.

This matches the explicit "끝까지 다 해" approach: keep moving forward;
don't block the whole v21 sequence on access to a Linux host.
