## §2 Medium Findings (기술적 부채 기록)

### H2: BridgeClient recreated per-request (MED)
- **Location**: `http_server.rs:316`
- **Issue**: 매 MCP HTTP 요청마다 새 BridgeClient 생성 (새 reqwest::Client + token 재읽기)
- **Impact**: 성능 — 현재 scale (1.6 req/s) 에서는 문제 없음
- **Fix**: V27에서 Arc<BridgeClient>를 AppState에 저장
- **Risk**: LOW (현재 트래픽에서 무시 가능)

### H3: Tool dispatch bypasses param validation (MED)
- **Location**: `http_server.rs:253-291`
- **Issue**: HTTP dispatch는 raw JSON args 전달, stdio는 #[serde(deny_unknown_fields)] 사용
- **Impact**: Typo field가 silently ignored
- **Fix**: V27에서 같은 param struct 재사용
- **Risk**: LOW (auth 뒤에 있고, 기능적으로는 동작)

### H4: notifications/initialized 응답 반환 (MED)
- **Location**: `http_server.rs:137`
- **Issue**: JSON-RPC 2.0 spec상 notification은 응답 없어야 함
- **Impact**: Spec non-compliance, 실제 MCP 클라이언트는 id 포함하므로 OK
- **Fix**: id 없으면 응답 skip
- **Risk**: VERY LOW

### D1/R1: TOCTOU race on peer file save (MED)
- **Location**: discovery.rs, register.rs:run_add/run_remove
- **Issue**: PeerLock 없이 load→mutate→save
- **Impact**: 동시 musu peer add 시 마지막 write wins
- **Fix**: PeerLock 사용
- **Risk**: LOW (CLI 수동 명령, 동시 실행 극히 드묾)

### D2: Corrupt cache silently ignored (MED) → **FIXED**
- tracing::warn 로그 추가 완료

### D5/R2: No addr validation (MED) → **FIXED**
- validate_peer_addr() 함수 추가 + run_add/run_remove에서 호출
