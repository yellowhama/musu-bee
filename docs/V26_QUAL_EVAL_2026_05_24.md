## 🧐 3대 아키텍처 성과 평가 (Architectural Breakdowns)

### 1. 분산 SQLite 스플릿 브레인 완벽 회피 (Database-Free State Tracking)
- **기존 문제**: 합의 알고리즘(Raft 등)이 결여된 다중 기기 환경에서 SQLite 로컬 DB를 상호 동기화하려 할 경우, 네트워크 유실이나 동시 쓰기 시 데이터 잠금(Locking) 및 이력 분기 충돌(Split-Brain)이 필연적이었습니다.
- **개선 성과**: 태스크의 세부 상태를 `~/.musu/tasks/{task_id}.json` 파일 단위로 관리하는 **플랫 파일 SSOT 모델**로 패러다임을 바꿨습니다. 최초 요청한 Originator 노드가 파일의 영구적 소유권을 갖고, 원격 실행 노드는 단방향 HTTP API 콜백 및 파일 갱신만 수행하여 분산 일치 복잡도를 회피하고 성능 병목을 제거했습니다.

### 2. 가상 네트워크 컨텍스트 전송 통합 (W14 Context Transport Baseline)
- **기존 문제**: A 기기에서 B 기기로 태스크를 Forwarding할 때, B 기기의 작업 디렉토리(cwd)에 소스 코드나 지식 파일이 없어 오작동하는 치명적인 워크스페이스 격차가 존재했습니다.
- **개선 성과**: `musu-brainai` 내에 파일 쓰기/읽기 API(`/api/files/*`)를 선결적으로 완비하여, 태스크 위임 시점에 워크스페이스 코드를 경량 압축(Zip)하여 원격으로 전송하고, 격리 실행 후 결과물을 회수하는 **경량 컨텍스트 전송(Context Transport)**의 완전한 인프라 기틀을 세웠습니다.

### 3. 단순 인덱서에서 "다이내믹 액터 성단"으로의 도약 (Galaxy V2 UI/UX)
- **기존 문제**: 이전의 Galaxy UI는 크롤링한 웹 문서, 유튜브 대본, Arxiv 논문 등 정적인 지식 조각들의 연관관계만 보여주어 실시간 기기 구동 상태를 알기 어려웠습니다.
- **개선 성과**: 지식 문서(Blue/Yellow/Red) 성단 주위에 **에이전트 실행 태스크(Green: running / Yellow: pending / Gray: completed) 노드**와 이들을 실행 중인 **물리 PC 기기 노드(Purple)**, **회사 프로젝트 노드(Pink)**를 D3.js 포스 레이아웃 상에서 시각적 선(Link)으로 실시간 엮어냈습니다. 이를 통해 지식의 흐름과 함대(Fleet)의 리소스 분포를 직관적으로 이해할 수 있는 프리미엄 제어판을 제공합니다.

## 🔒 보안 및 성능 고도화 검증

- **평문 토큰 누출 게이트 (🔐 C-SEC-8)**: Tailscale 가상 IP망(`100.64.0.0/10`)을 벗어나 평문 HTTP 폴백 시 Bearer 토큰이 외부에 노출되어 원격 코드 실행(RCE)을 당할 수 있는 위협을 확인했습니다. 향후 P2P 인증은 **Ed25519 서명 기반 챌린지-응답 메커니즘**으로 무장할 것을 마스터 플랜에 확정했습니다.
- **윈도우 자원 점유 최적화 (💻 Zero-CPU Heartbeat)**: 60초 주기 하드웨어 메타데이터 수집을 위해 무거운 PowerShell/wmic 프로세스를 주기적으로 띄우는 배터리 방전 부채를 식별했습니다. Windows 환경에 한해 `winreg` 크레이트를 이용한 **레지스트리 직접 조회 FFI 구현**을 확정하여 하트비트 스파이크를 Zero 수준으로 감축시킵니다.
