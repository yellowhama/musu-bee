W13: MCP HTTP+SSE Server Endpoint

What shipped

  • control/http_server.rs — JSON-RPC 2.0 over HTTP (same 14 tools as stdio MCP)
  • /mcp/v1/messages POST — initialize, tools/list, tools/call dispatch
  • /mcp/v1/sse GET — SSE heartbeat stream (30s interval)
  • /mcp/v1/health GET — server health + tool count
  • Bearer auth shared with bridge (axum middleware)
  • Loopback-only default

Security considerations

  • MCP routes go through existing bridge auth middleware (require_bearer)
  • Loopback-only default — external bind requires explicit --mcp-bind-external opt-in
  • JSON-RPC 2.0 version validation before processing
  • Tool dispatch mirrors stdio exactly — no separate code path for HTTP
