## §5 Risks + mitigations

| Risk | Severity | Mitigation |
|---|---|---|
| R1: XSS via markdown→HTML | HIGH | W-1 XSS test vector (script tag injection, onclick, javascript: URLs); DOMPurify default config; W-7 Playwright assertion that script tags from markdown source don't execute |
| R2: react-markdown bundle size impact | MED | react-markdown gzipped 13KB. Acceptable. Measure musu-bee build size before/after in W-1 closure |
| R3: view_count race condition under concurrent reads | LOW | SQLite UPDATE is atomic. If contention emerges, optimistic counter (no SELECT-then-UPDATE — per [[pattern-toctou-atomic-update]]). Acceptable lossy counting for ROI evidence |
| R4: H-2 try/catch breaks existing client error contract | MED | Document new 500 + JSON error shape in handler comments. Test that pre-existing 200/4xx paths unchanged. Per V23.4 F-B2-3 precedent, no known client pattern-matches on old error shape |
| R5: H-4 feature flag forgotten in env config | LOW | Default OFF + log warning if mesh forward called without flag set + propagation skipped. Documented in CHANGELOG and INSTALL.md |
| R6: H-4 QUIC sidecar path also breaks request_id chain | MED (Phase 0 surprise) | Phase 0 noted `_forward_quic()` has same gap. H-4 scope expanded to cover both HTTP + QUIC paths. Or QUIC deferred to V23.6 if testing complexity high. Decision in H-4 detail plan. |
| R7: W-6 manual HTML mirror diverges from markdown source | MED | W-6 ingest script reads markdown → renders → stores in wiki_pages.source_html (or generates standalone HTML mirror file). Markdown remains source of truth. Re-run script regenerates. Document re-run procedure in W-8 runbook |
