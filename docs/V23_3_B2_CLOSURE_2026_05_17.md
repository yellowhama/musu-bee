## ¬ß1 What shipped

B2 lands a single **unauthenticated** POST endpoint `POST /v1/telemetry/install_attempt`
on `signaling.musu.pro` (musu-relay), backed by a new SQLite table at schema **v42**
(`install_attempt`), with in-memory per-instance token-bucket rate-limiting keyed
on `(install_id, source_ip)`. The Windows PowerShell installer (`install-wsl2.ps1`)
emits one POST per pre-bootstrap failure step across 10 trigger sites (16 sub-site
emits matching the existing `Save-MusuFailureDump` granularity), letting the
operator collect failure telemetry from hosts that never reach a working musu
install (i.e., never obtain an `account_key`, so the existing HMAC-authed
`/v1/telemetry/install` route is unreachable to them).

### 1.1 Server-side surface (Builder commit `770c588`)

- **Schema v42** `install_attempt` table ‚ÄĒ DDL at `musu-relay/src/signaling/telemetry.ts:154` (`MIGRATION_V42_INSTALL_ATTEMPT` constant), shape mirrors the v40 `telemetry_install` template exactly (INT PK + `received_at` + `musu_install_id` + step/error_class + `elapsed_ms` + optional context columns + `schema_version` default 1).
- **`applyMigrations()` v42 conditional + env-gate** at telemetry.ts:216-246. Mirrors v41 pattern: in `NODE_ENV=production` without `MUSU_TELEMETRY_V42_AUTHORIZED=1` set, the v42 block throws before applying DDL; in test env (`NODE_ENV !== "production"`) the env-gate is bypassed so CI runs migrations cleanly.
- **In-memory token-bucket rate-limit** module at telemetry.ts:295-405 ‚ÄĒ Map<string, BucketState> keyed on `${install_id}:${source_ip}`, 20 tokens/hr refill, `RL_MAP_HARD_CAP=10000`, `RL_REAPER_MAX_AGE_MS=10min` (telemetry.ts:322), reaper frequency 1min, **LRU eviction at cap** (telemetry.ts:354-360 ‚ÄĒ evicts single oldest entry by min `lastRefillMs` before insert, per Critic C-B2-M5 ship-with-evict not refuse).
- **`POST /v1/telemetry/install_attempt` route handler** inside `makeTelemetryRouter()` ‚ÄĒ handler body at telemetry.ts:830-920 (no auth middleware; opts out of `requireTelemetrySecret`); field-validation calls `optClamp(...)` with char-class regex enforcement per Critic C-B2-M1 (telemetry.ts:285-291 ‚ÄĒ `INSTALL_ID_RE`, `STEP_RE`, `ERROR_CLASS_RE`, `OS_VERSION_RE` post-audit-fix1, `BIOS_VT_RE`, `HOST_CLASS_RE`, `INSTALLER_VERSION_RE`); range check `0 <= elapsed_ms <= 24*60*60*1000` per Critic C-B2-M2.
- **`app.set("trust proxy", 1)`** at `musu-relay/src/signaling/server.ts:324` (immediately after `const app = express()`), so `req.ip` is the real client IP, not the Fly edge proxy IP, per Critic C-B2-H1. **Positive defensive side-effect**: `/install`, `/nat_pierce`, `/agent_spawn` also see real client IPs if any future code path reads `req.ip` ‚ÄĒ explicitly accepted documentation in ¬ß11 of wiki/390.
