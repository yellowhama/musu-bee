## В§7 Const VI experiment record

**Question**: does the V23.3 B6 recipe produce a byte-identical `musu-backend.tar` across two intra-hour builds on the same Alpine 3.19 WSL2 host?

**Setup**: Alpine 3.19 WSL2 distro `alpine-musu-build`; `apk add tar binutils` for GNU tar + strip/objcopy; manifest pins fixed; SDE = `git log -1 --format=%ct HEAD` = (commit timestamp of `89e07a1`).

**Method**:
1. Build A: `bash build-musu-backend.sh --arch amd64 --k3s-version v1.30.4 --output /tmp/musu-backend-b6-A.tar`
2. ~10 min later, Build B: `... --output /tmp/musu-backend-b6-B.tar`
3. `sha256sum` both

**Result**: identical sha256 `ace155f730b2402c31ec8d65acd149f92c99ceca3b5b0448d4ad890551aedf7e`.

Root-caused via `tar -x` + `diff -qr` to `node_modules/better-sqlite3/build/` intermediates (Makefile, .o, .a). Added step 3.c to prune them.
