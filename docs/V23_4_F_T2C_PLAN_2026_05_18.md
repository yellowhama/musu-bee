### §2.4 AddPcWizard — inline modal component (within `fleet/page.tsx`)

Per OQ-CRIT-2 (resolving C-T2C-3): AddPcWizard fetches `GET ${BRIDGE_URL}/api/agents` on modal open to populate the multiselect from real bridge data. Fields:

- `url` (string, required, e.g. `http://192.168.1.10:8070`)
- `agents[]` (multiselect populated from `GET /api/agents`; 503 fallback to free-text)

**State**:
- `agentsList: Array<{value: string, label: string}> | null` — populated on modal open
- `agentsLoading: boolean`
- `agentsFallback: boolean` — true when fetch fails, switches input mode to free-text

**Success path — `fetchAgents()` helper** (called on modal open via `useEffect([open])`):

```ts
async function fetchAgents() {
  setAgentsLoading(true);
  try {
    const res = await fetch(`${BRIDGE_URL}/api/agents`);
    if (!res.ok) throw new Error(`${res.status}`);
    const data = await res.json();
    // data: Array<{id: string, name: string}>
    setAgentsList(data.map((a: any) => ({ value: a.id, label: a.name })));
    setAgentsFallback(false);
  } catch (e) {
    setAgentsList(null);
    setAgentsFallback(true);  // banner: "agents list unavailable, enter comma-separated ids"
  } finally {
    setAgentsLoading(false);
  }
}
```

**Success-path UI shape** (when `agentsList` present):
```tsx
<select multiple value={agents} onChange={...}>
  {agentsList.map(a => <option key={a.value} value={a.value}>{a.label}</option>)}
</select>
```

**Fallback-path UI shape** (when `agentsFallback === true`):
```tsx
<div style={{ background: "var(--warn-bg)" }}>agents list unavailable, enter comma-separated ids</div>
<input type="text" placeholder="agent-id-1, agent-id-2"
  onChange={e => setAgents(e.target.value.split(",").map(s => s.trim()).filter(Boolean))} />
```
