### Â§2.2 `src/components/fleet/FleetClient.tsx` â€” main client component

Mirrors `/m/[id]/page.tsx` structure exactly (F-R1):

- `useEffect` + 5s `setInterval` polling `GET ${BRIDGE_URL}/api/machines`
- Dual `EventSource` subscriptions: `?source=machines` + `?source=machine_capacity` (F-R3)
- State: `machines: FleetMachine[]`, `error: string | null`, `wizardOpen: boolean`
- Renders: header strip + grid of `<PcCapacityCard machine={m} />` + `<AddPcWizard open={wizardOpen} ...>` + error banner
- Cleanup: `clearInterval` + `EventSource.close()` on unmount
- BRIDGE_URL: `process.env.NEXT_PUBLIC_MUSU_BRIDGE_URL || "http://localhost:8070"` (F-R13)

Type definition (top of file):

```ts
type FleetMachine = {
  id: string;
  hostname: string;
  os: string;
  arch: string;
  status: "online" | "offline" | "stale";
  last_seen_at: number;

    gpu_vram_total_mb: number;
    mem_used_mb: number;

```

No client-side staleness threshold computation (F-R9) â€” `machines.status` from bridge is authoritative.

### Â§2.4 `src/components/fleet/AddPcWizard.tsx`

- `agents[]` (multiselect â€” see Â§9 Q3 below)

