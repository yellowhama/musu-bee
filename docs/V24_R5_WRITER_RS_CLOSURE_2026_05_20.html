§1 What R5 was

R5 is the first sub-workstream of R-cleanup under the V24 Rust-Big-Bang master plan (wiki/490), promoted to position #1 by the operator pivot at GOAL.md §A.1.1 on 2026-05-20 evening. The pivot reordering ("single-machine completeness before R9") moved R5 ahead of R6/R3/R4 because the writer-stub gap was load-bearing for every agent task on the freshly-installed Rust binary.

The gap itself was surfaced by R8 closure (wiki/498c §8): on the 4060Ti host, after the legacy Python install was torn down per operator authorization, the native Rust POST /api/companies/{id}/run handler still INSERTed a route_executions row with status pending_python_writer and tried to forward the work to a now-dead Python facade on :8071. The forward predictably returned error sending request; the row got marked python_unreachable; the endpoint still returned 202 with a task_id, but no work ever happened. Same shape for POST /api/tasks/delegate in handlers/tasks.rs:113-161.

R5 replaces both stubs with native Rust agent-task execution: tokio::spawn-managed task lifecycle, tokio::process::Command driving the claude CLI subprocess with line-buffered stream-json parsing, tokio::sync::broadcast(100) for SSE fan-out on GET /api/tasks/events, and a new DELETE /api/tasks/{task_id} cancel endpoint. Schema goes v1 → v2 with 6 NULLable additive columns on route_executions. Memory tag [[feedback-no-python]] is materially satisfied: under R5, no normal operation touches the Python facade anymore (R-cleanup R3/R4/R6 close remaining surface).

§5.1 Deviation: Builder D1 — per-column-skip idempotency

Plan §4.2 prescribed all-or-nothing transaction semantics for apply_v2. Builder shipped per-column-skip idempotency via pragma_table_info pre-check in core/migrate.rs apply_v2: each ALTER TABLE statement is preceded by a query against pragma_table_info('route_executions') and skipped if the column already exists. The all-or-nothing tx wrapper is retained around the loop.

Auditor A1 ACK: this is a safer evolution. Plan §4.2 explicitly acknowledged concurrent-boot races (R5-W4); the per-column skip is the natural Rust idiom for an idempotent ALTER ladder and gracefully handles the case where a partial migration was interrupted mid-loop on a prior boot (one ALTER applied, then a crash before set_version(2) committed). Plan-intent (idempotent additive migration, no row rewrite, atomic version bump) preserved.
