# MUSU SSOT (1-page) — 2026-04-09

## 한 줄 정의

**MUSU는 1개 이상의 컴퓨터를 하나의 작업공간으로 묶고, 그 위에서 다수의 회사를 굴리며, 하네스 자동화 + 에이전트 고도화를 통해 스스로 발전하는 시스템이다.**

3 동사 / 3 단계 / 3 layer:

1. **묶는다** (substrate) — 1+ PC → 1 workspace. mesh router + nodes.toml + Tailscale + sqlite + Rust binary. V21-V24 ship.
2. **굴린다** (organization) — 1 workspace 위에 N companies (CEO/Engineer/QA agents + 회사 memory + 회사 KPI). V21 `companies` table + V21.F UI ship.
3. **발전한다** (improvement) — Domain A (하네스 자동화: musu 가 musu 안에서 self-improvement task 생성/실행) + Domain B (에이전트 고도화: 회사 안 agents 가 시간 지나면 더 잘하게 됨 — instruction file self-update + 회사 memory pattern 학습 → adapter context inject). V26+ thesis. partial implemented (agent-team mode + V25-OPS codification).

기본 인터페이스는 `CLI/MCP`이고, 채팅/웹 GUI는 `musu.pro`에서 **어디서든 접속 가능한 원격 운영면(surface)**이다.

이전 한 줄 ("control plane으로 묶는") = 단계 1 만 표현. V26+ thesis 가 단계 2+3 까지 포괄하도록 reshape (2026-05-21 user grill session, wiki/507 §9 참조).

3 산업 표준 (HPC GPU mesh / SSI cluster / VM clustering) 은 모두 단계 1 차원의 product. musu 는 단계 1 을 다른 방식 (node-aware) 으로 한 다음 단계 2+3 까지 가는 것이 thesis — wiki/507 + `[[feedback-no-yagni-architecture]]` + `[[decision-musu-3tier-thesis]]` 참조.

## 제품 구조(제품 언어)

**Core concerns**
- Control: 상태를 읽고(queue/lane/run/report), 실행을 지시하고(scheduler/executor/watchdog/supervisor), 위험은 승인으로 분리한다(approval).
- Connectivity/Trust: 멀티노드에서 “신뢰 가능한 연결/증거”를 남긴다(trust level, evidence, artifacts).

**Interfaces**
- Default: `Codex CLI` / `Claude Code` / `MCP`
- Pro: `musu.pro` Web GUI(원격 관측/지시/이어받기; 필요 시 채팅형 UX)

## Windows distribution note (2026-05-27)

Windows 배포는 이제 하나의 경로로 읽으면 안 된다. 현재 제품 계약은 3갈래다.

- **Current operator path**: direct-download bootstrap (`install.ps1`, `~/.musu/bin`, platform service registration)
- **Packaged validation path**: local sideload / MSIX manual bridge runtime
- **Target reviewed path**: Microsoft Store / restricted-capability auto-start runtime

이 셋은 같은 설치 계약이 아니다. Packaged build 는 package-managed install/update 를 따라야 하며, binary self-copy, PATH mutation, Task Scheduler registration, GitHub self-update 를 제품 기본 가정으로 두면 안 된다.

2026-05-27 state lock:

- **local sideload / manual bridge**: repo-local 검증 완료
- **Store-reviewed auto-start**: artifact / submission bundle 준비 완료, Microsoft Partner Center verification + review 대기
- **즉 현재 Windows packaged truth** = auto-start 없는 local sideload 는 완성, auto-start 있는 Store path 는 외부 승인 blocker 보유
