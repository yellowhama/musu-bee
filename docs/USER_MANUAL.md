# MUSU User Manual

## 1. 노드(Node) 연결 및 관리
- **연결 방법**: 좌측 패널의 `Connected Devices` 메뉴에서 연결된 기기 목록을 확인합니다.
- 새로운 기기를 연결하려면 해당 기기에서 `musu-rs` 데몬을 실행해야 합니다.
- 목록에서 원하는 기기를 클릭하면, 중앙 뷰어의 제어 타겟이 해당 기기로 변경됩니다. 여러 기기를 다중 선택하여 브로드캐스트(Broadcast) 명령을 내릴 수도 있습니다.

## 2. 모드별 상세 가이드

### 2.1. Dev Mode (개발자 모드)
이 모드는 기기의 파일 시스템과 터미널에 직접 접근할 때 사용합니다.
- **Split View**: 좌측에는 파일 탐색기, 우측에는 터미널이 열립니다.
- 파일 탐색기에서 파일을 더블클릭하면 즉시 내용을 확인하거나 수정할 수 있으며, 드래그앤드롭을 지원합니다.
- 우측 채팅창에서 코딩 관련 질문을 하면 터미널에 명령을 입력 스크립트로 제공합니다.
- **상태 보존**: 다른 모드에 다녀와도 터미널 커서의 위치나 열어둔 파일 탐색기의 경로는 사라지지 않고 그대로 유지됩니다.

### 2.2. Town Mode (타운 모드)
작업 중인 AI 에이전트들의 상태를 게임 화면처럼 직관적으로 볼 수 있습니다.
- 특정 로봇 아바타를 클릭하면, 해당 에이전트의 성향, 롤(역할), 시스템 프롬프트를 수정할 수 있는 사이드바가 열립니다.
- 에이전트의 이름이나 외형을 커스터마이징하여 나만의 AI 팀을 꾸려보세요.

### 2.3. Butler Mode (집사 모드)
가장 심플한 모드입니다.
- 화면 중앙의 메인 집사에게 우측 채팅창을 통해 원하는 것을 말하세요.
- 예: *"오늘 배포된 빌드 로그 좀 분석해서 요약해줘."*
- 집사가 비동기로 백그라운드 분석을 진행하며, 완료되는 즉시 원형 차트나 요약 위젯으로 예쁘게 띄워줄 것입니다.

## 3. 실시간 제어와 Generative UI
- MUSU의 챗봇은 일반적인 텍스트 챗봇과 다릅니다. 사용자의 요구에 맞는 UI 요소(그래프, 로딩바, 터미널 로그)를 제공받는 홀로그램 알림을 띄워줍니다.
- **비동기 푸시 오케스트레이션**: 예를 들어 *"10분 동안 서버 상태를 모니터링 해줘"* 라고 명령하면, 챗봇은 알겠다고 대답한 후 조용해집니다. 그리고 10분 뒤에 스스로 결과를 화면 우측 상단 위젯 스택으로 푸시(Push)하여 결과를 알려줍니다. 채팅창 안에만 머무르지 않는 진정한 에이전트 경험을 느껴보세요.
