# MUSU Configuration Reference

All environment variables used by musu-bridge, musu-worker, musu-core, and related services.

Set variables in `~/.musu/bridge.env` (created by `scripts/install.sh` or `musu install`) or export them before starting services.

Windows distribution note:
- Direct-download builds and Store/MSIX packaged builds share `~/.musu/` runtime state.
- They do **not** share the same install/update contract.
- Store/MSIX builds must not assume `~/.musu/bin` is the authoritative install source or that MUSU is allowed to self-update its own binaries.

---

## Bridge

| Name | Default | Required | Description |
|------|---------|----------|-------------|
| `MUSU_BRIDGE_TOKEN` | *(none)* | **Yes** (production) | Bearer token for all bridge API calls. Auto-generated by `install.sh`. |
| `MUSU_DISTRIBUTION` | `auto` | No | Runtime distribution mode. `auto` detects packaged Windows builds, `direct-download` forces bootstrap/service semantics, `store-msix` disables MUSU-managed self-update and platform-service install and expects packaged binaries. |
| `BRIDGE_HOST` | `127.0.0.1` | No | Bind address. Set to `0.0.0.0` for LAN/Tailscale access. |
| `BRIDGE_PORT` | `8070` | No | Bridge HTTP port. Local CLI / control clients prefer `~/.musu/services/bridge.json` when present, then fall back to this port. |
| `MUSU_BRIDGE_PUBLIC_URL` | *(empty)* | No | URL advertised to peers during pairing (e.g. `http://100.64.0.1:8070`). |
| `MUSU_BRIDGE_ALLOWED_ORIGINS` | `https://musu.pro` | No | Comma-separated CORS origins for browser clients. |
| `MUSU_BRIDGE_LOCALHOST_AUTH` | *(unset)* | No | If set, localhost requests also require Bearer token (default: localhost bypasses auth). |
| `MUSU_NODE_NAME` | hostname | No | Human-readable name for this node in the registry and UI. |
| `MUSU_NODE_ROLE` | `primary` | No | Node role (`primary` or `secondary`). Controls CEO heartbeat scheduling. |
| `MUSU_ENV` | `development` | No | `production` enables stricter security checks. |
| `MUSU_PLAN` | `free` | No | `pro` unlocks premium features. |
| `LOG_LEVEL` | `INFO` | No | Python log level: `DEBUG`, `INFO`, `WARNING`, `ERROR`. |
| `MUSU_DB_PATH` | `~/.musu/musu.db` | No | Path to the musu-core SQLite database. |
| `MUSU_MAX_CONCURRENT_TASKS` | `20` | No | Maximum concurrent task executions on the bridge. |
| `MUSU_CHANNEL_MAX_TASKS` | `5` | No | Maximum concurrent tasks per channel. |
| `MUSU_SEMAPHORE_ACQUIRE_TIMEOUT_SEC` | `60` | No | Seconds to wait for a concurrency slot before rejecting. |
| `MUSU_DISPATCH_HASH_TTL_SEC` | `120` | No | Dedup TTL for dispatch hashes (prevents duplicate task execution). |
| `MUSU_DISABLE_RATE_LIMIT` | `0` | No | Set to `1` to disable rate limiting entirely. |
| `MUSU_BEE_URL` | `http://localhost:3001` | No | URL of the musu-bee Next.js frontend. |
| `MUSU_PRIMARY_URL` | *(empty)* | No | URL of the primary node (used by secondary nodes for sync). |

## nodes.toml Format Reference

Location: `~/.musu/nodes.toml` (auto-generated by `install.sh`).

```toml
[mesh]
self = "my-pc"                  # Must match one of the [[mesh.nodes]] names
worker_port = 9700              # Default worker port for all nodes
health_interval_sec = 30        # Seconds between health checks to peer nodes

[[mesh.nodes]]
name = "my-pc"                  # Unique node identifier
machine = "my-pc-desktop"       # Human-readable machine label
os = "linux"                    # Operating system: linux, darwin, windows
tailscale_ip = "100.64.0.1"    # Tailscale IP (leave empty for local-only)
url = "http://100.64.0.1:8070" # Bridge URL for this node
gpu = "RTX 4060"                # GPU description (empty if none)

[[mesh.nodes]]
name = "macbook"
machine = "macbook-air"
os = "darwin"
tailscale_ip = "100.64.0.2"
url = "http://100.64.0.2:8070"
gpu = ""
```

### Key rules

- `mesh.self` must exactly match one `[[mesh.nodes]].name` entry.
- `url` should use the Tailscale IP for cross-network access, or `127.0.0.1` for local-only.
- `worker_port` is the default; individual nodes use whatever port their musu-worker actually listens on.
- The file is auto-generated by `install.sh` with auto-detected GPU, OS, and Tailscale IP.
