§1 What R6 was

R6 restores the operator-ergonomic surface deleted at R8 start. Per wiki/498c §3 "the install-clean delta", 11 systemd user units were removed when the legacy Python install was wiped to free :8070 for the Rust bridge. R6 brings back the load-bearing subset — implemented as native Rust subcommands on the single musu binary, with platform-service registration and a hybrid auto-update flow.

Six new subcommands ship on musu: install, uninstall, auto-update, supervise (wraps existing musu-supervisor-core via musud), schema-precheck, apply-schema. Operator can now do musu install on a fresh machine, get a Scheduled Task (Windows) / systemd user unit (Linux) / launchd LaunchAgent (macOS) that auto-starts the bridge on logon, and an auto-update timer that pulls signed binaries from GitHub releases with rollback safety on /health failure.

This closes the operator question that drove the §A.1.1 R-cleanup pivot: "자동 업데이트 기능 깉은 편의성 기능 있는지" — yes, now.

  • QB5 --i-have-a-backup — install/uninstall.rs: new CLI flag; confirm_purge bails when db_recent && !i_have_a_backup.
  • QB6 system_update auth test — handlers/system_update.rs::tests: asserts 401 without bearer via tower oneshot.
  • QA1 schema_gate path bug — install/schema_gate.rs:113: replaced Path::with_extension(format!(...".new")) (which produces <name>..new double-dot on Unix) with staged_swap::with_suffix(&target, ".new")?. Made with_suffix pub(crate).
