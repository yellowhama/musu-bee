# MUSU 1.15.0-rc.1 Second-PC Runtime CPU Subrole Import Gate

Date: 2026-06-06 KST
Status: Done with concerns. This hardens second-PC evidence import and operator
handoff. It does not close the second-PC, hosted P2P, support mailbox, or Store
release blockers.

## Summary

The previous runtime CPU subrole gate made fresh CPU evidence separate
`bridge_runtime`, `desktop_shell`, and `webview2_helper`, but the second-PC
return importer still mostly trusted the returned release-check summary and
file presence. A stale return zip generated by an older wrapper could therefore
look structurally complete to `import-second-pc-return.ps1
-RequireReleaseGateEvidence` even if the CPU JSONs lacked the current subrole
contract.

That gap is now closed. Release-gate second-PC imports require current CPU
subrole evidence directly in the imported idle CPU and runtime matrix JSON, and
the second-PC release-check JSON must report
`runtime_cpu_subrole_contract_ok=true`.

## Code Changes

- `scripts/windows/run-second-pc-release-check.ps1`
  - adds subrole-summary checks for idle CPU and scenario matrix measurements
  - writes `runtime_idle_cpu_subrole_summary`
  - writes `runtime_cpu_scenario_subrole_summary`
  - writes `runtime_cpu_subrole_contract_ok`
  - prints bridge/runtime/desktop/WebView2 subrole counts in human output
- `scripts/windows/import-second-pc-return.ps1`
  - directly parses imported `musu.runtime_idle_cpu_evidence.v1`
  - directly parses imported `musu.runtime_cpu_scenario_matrix.v1`
  - rejects release-gate imports missing `process_counts_by_subrole`,
    `max_one_core_percent_by_subrole`, `memory_totals_by_subrole_mb`,
    `cpu_attribution` subrole fields, or `process_subrole` rows
  - requires `bridge_runtime`; requires `desktop_shell` for desktop/startup
    evidence; requires `webview2_helper` for `desktop-open`
  - rejects stale release-check JSON that lacks
    `runtime_cpu_subrole_contract_ok`
- `scripts/windows/prepare-multidevice-test-kit.ps1`
  - documents that old return zips without subrole fields are diagnostic only
- `scripts/windows/prepare-operator-action-pack.ps1`
  - adds runtime idle CPU and runtime matrix files to return instructions
  - calls out `runtime_cpu_subrole_contract_ok=true`
- `scripts/windows/prepare-final-operator-gate-packet.ps1`
  - updates final handoff instructions with the new import contract
- `scripts/windows/verify-operator-action-pack.ps1`
  - fails operator packs whose quickstart/kit README omit the subrole contract
- `scripts/windows/verify-final-operator-gate-packet.ps1`
  - fails final packets whose README/importer/kit checks omit the subrole
    contract
- `scripts/windows/test-release-evidence-verifiers.ps1`
  - adds a source-contract regression case for second-PC import subrole gating

## Product Spec Impact

The product boundary is unchanged:

- MUSU Desktop is the local executor on each Windows device.
- MUSU.PRO is remote input, project/company room, rendezvous, path-selection,
  relay-fallback policy, and evidence/control coordination.
- MUSU.PRO still must not become the execution server or default payload data
  path.

The evidence contract changed:

- A second-PC return zip is release-grade only if it was generated by the
  current wrapper and includes `runtime_cpu_subrole_contract_ok=true`.
- Older second-PC return zips can still be used for diagnosis, but they cannot
  satisfy `-RequireReleaseGateEvidence`.
- A real second Windows PC still has to run the current kit to close the
  second-PC idle CPU, runtime matrix, and multi-device route gates.

## Qualitative Evaluation

Local single-machine beta remains strong, but public desktop release is still
not ready.

- Local desktop/runtime path: high confidence for one-machine execution.
- Second-PC evidence flow: stronger after this change because stale CPU returns
  are rejected instead of quietly imported.
- Hosted P2P release proof: still blocked; the repo still lacks release-grade
  relay payload transport proof and live owner-scoped `musu.pro` evidence.
- Operator risk: lower; the quickstart now says exactly which CPU subrole fields
  make a return zip release-current.

Code audit finding fixed in this pass:

- Medium: `import-second-pc-return.ps1 -RequireReleaseGateEvidence` trusted the
  returned release-check booleans too much and did not directly enforce the new
  CPU subrole contract. Fixed by parsing imported CPU evidence and requiring
  `runtime_cpu_subrole_contract_ok`.

No remaining high or medium issue was found in the changed scripts after the
regression run.
