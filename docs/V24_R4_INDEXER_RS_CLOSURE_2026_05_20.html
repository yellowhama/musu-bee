§1 What R4 was

R4 replaces Python musu-indexer (3,115 LOC + 252-line Go scanner binary) with native Rust per-workspace file/code indexing. Substrate: SQLite FTS5 (already in tree via sqlx; tantivy YAGNI per [[feedback-no-yagni-architecture]]). Per-workspace <work_dir>/.musu_dev.db file matches Python parity — no Const III gate on main musu.db.

Scope sharpening (F2): Python musu-indexer is a hybrid — 1,500 LOC of session/spy/PTY tooling + ~1,100 LOC of actual indexer logic. R4 ports ONLY the indexer logic. session/spy/PTY surface is DROPPED (zero non-test usage signal; R10 deletes alongside Python).

R4 is the FINAL native module — all 4 R-cleanup native crates (R3 control, R4 indexer, R5 writer, R6 installer) are now shipped. Remaining V24 work: R10 (Python deletion + V24 closure + CHANGELOG 1.14.0 + §9.12 operator-attested gate).
