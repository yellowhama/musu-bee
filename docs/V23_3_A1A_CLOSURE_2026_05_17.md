## §7 Forward-pointers for A1.b (Pod spec)

Recommendations consolidated from security-engineer HANDOFF NOTES + Critic resolution:

- `securityContext.runAsUser: 1000`, `runAsGroup: 1000`, `fsGroup: 1000` — matches `/etc/passwd:musu:x:1000:1000::/home/musu:/sbin/nologin`
- `securityContext.runAsNonRoot: true` — fail-fast if image regresses
- `securityContext.allowPrivilegeEscalation: false` — defense-in-depth for inherited SUID binaries (S-A1a-M2)
- `securityContext.readOnlyRootFilesystem: true` — neutralizes SUID exec; requires emptyDir for `/tmp` + PVC mount for `/home/musu/.musu/db/`
- `securityContext.capabilities.drop: ["ALL"]` — bridge needs zero Linux caps
- `terminationGracePeriodSeconds: 45` — 30s task drain + 10s controller stop + 5s buffer per `server.py:920-941`
- ConfigMap/Secret mount path: `/home/musu/.musu/account_key` (Critic H2 resolution; HOME validated in audit)
- `MUSU_BRIDGE_TOKEN` Secret value MUST be ≥32 chars — now enforced at container start time by `server.py:208-217` (post-S-A1a-H2 fix); A1.b ConfigMap/Secret construction script should still reject <32-char tokens defensively
- liveness probe: `/health` returns JSON with key `worker` (NOT `worker_ok` — plan §10.3 had spec drift; corrected in B-A1.a-5)
