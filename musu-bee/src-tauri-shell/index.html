    Looking for local Private Mesh evidence.

    1. Control host

      Generate your self-hosted Headscale + Caddy HTTPS + embedded DERP bundle. No command to copy — give it your mesh host and click.

      • Start control host

        Bring the control plane online from here — MUSU runs docker compose and confirms Headscale is healthy. (Requires Docker on this machine.)

        For a public mesh, point your domain at this host and confirm HTTPS before adding remote PCs: scripts/check-public-endpoint.ps1.

      • Device-add pass

        With the control host running, issue a one-use MUSU device-add pass from this cockpit. MUSU writes the secret-bearing pass under device-add-passes/ and only shows the file path to copy to the target PC.

        Pass schema musu.device_add.v1

      • Every PC

        Copy that generated pass file to each target PC, then join with the pass path. MUSU rechecks the public /health endpoint before marking the control server verified and consumes the secret-bearing file after a successful join.

        musu mesh join --device-add-pass <musu.device_add.v1.json>
        musu mesh verify --target-ip <peer-100.x.y.z>
        musu mesh physical-peer-evidence --json
      • Release proof

        Copy the physical-peer evidence JSON and its `.sha256` sidecar from the target PC to the same folder on the source PC, then run release proof. Without that target-generated pair, MUSU shows only software route proof, not final release trust.

        musu mesh release-proof --target-node <node> --target-ip <peer-100.x.y.z> --expected-control-server-url https://mesh.your-domain --physical-peer-evidence <copied-target-pc-physical-peer-evidence.json> --json
