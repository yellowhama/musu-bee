    1. Control host

      Generate your self-hosted Headscale + Caddy HTTPS + embedded DERP bundle. No command to copy — give it your mesh host and click.

      • Start control host

        Bring the control plane online from here — MUSU runs docker compose and confirms Headscale is healthy. (Requires Docker on this machine.)

        For a public mesh, point your domain at this host and confirm HTTPS before adding remote PCs: scripts/check-public-endpoint.ps1.

      • Device-add pass

        With the control host running, issue a one-use MUSU device-add pass from this cockpit. MUSU writes the secret-bearing pass under device-add-passes/ and only shows the file path to copy to the target PC.

        Pass schema musu.device_add.v1

      • On the new PC

        Install MUSU on the target PC from musu.pro, copy the generated pass file over, then paste its path here and join — no command to type. MUSU re-checks the control server /health and consumes the secret file after a successful join.

        Once joined, the new machine appears in your fleet above and you can give it orders. Checking the connection and capturing proof is done for you — use the buttons in “Private Mesh proof” and each machine’s “Run proof”; the raw musu mesh commands stay under “Having trouble?”.

      • Release proof (optional)

        Want release-grade proof that the two machines really reached each other? Copy the target PC’s physical-evidence file over (its path goes in “Release evidence” above), then click Run release proof there. No command to type — MUSU runs it and shows a plain verdict. Skip this and the mesh still works; you just get software-route proof instead of release-grade.
