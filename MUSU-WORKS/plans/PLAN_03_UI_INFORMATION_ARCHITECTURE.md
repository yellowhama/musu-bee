# PLAN 03: UI Information Architecture

## 목적

회사와 프로젝트 운영 모델을 화면 구조로 내린다.

## 입력

- [HOW_A_COMPANY_SHOULD_BE_BUILT.md](/home/hugh51/musu-functions/MUSU-WORKS/HOW_A_COMPANY_SHOULD_BE_BUILT.md)
- PaperClip organization / project / pipeline 흐름

## 범위

### 회사 화면

- company overview
- org chart
- agent roster
- approvals queue
- policy center
- capability catalog
- project list

### 프로젝트 화면

- project overview
- attached agents
- task / pipeline board
- memory
- activity / audit stream
- outputs

## 완료 조건

- 회사 첫 화면과 프로젝트 첫 화면의 핵심 블록이 정해져 있다.
- navigation hierarchy가 정해져 있다.
- org chart와 approvals queue의 우선순위가 정해져 있다.
