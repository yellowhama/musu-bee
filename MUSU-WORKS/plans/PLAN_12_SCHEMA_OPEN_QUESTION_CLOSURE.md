# PLAN 12: Schema Open Question Closure

## 목표

현재 persistence draft에 남아 있는 open question을 닫고, role template / attachment / session 모델까지 반영한 schema decision note를 고정한다.

## 범위

- company memory 분리 여부
- policy baseline vs project override 구조
- human principal 모델
- project agent session persistence 추가 여부

## 현재 truth

- role template와 session 모델 문서는 생겼다.
- 기존 schema draft에는 session 테이블이 아직 없다.
- open questions가 문서 하단에 남아 있다.

## 작업 목록

1. session persistence 필요성 판정
2. human principal 모델 선택
3. company memory table 여부 결정
4. policy override shape 결정
5. decision note와 schema draft 반영

## 완료 기준

- schema draft의 open question이 decision으로 바뀐다.
- session persistence가 문서상 명시된다.
- final touch 문서에서 schema parity 체크가 가능한 수준이 된다.
