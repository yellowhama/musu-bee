# MUSU Execution Strategy

## 실행 원칙

1. 회사는 운영 방식만 가진다.
2. 프로젝트는 제품 내용만 가진다.
3. 에이전트는 직무 방식만 가진다.
4. 실제 운영으로 검증한 것만 제품 capability로 환원한다.

## 지금 당장 해야 하는 일

### Track 1. Company Runtime To Product

- `musu_corp`에서 검증된 company runtime surface를 제품 capability로 환원
- owner:
  - root control layer
  - root runtime capability
  - `MUSU-WORKS`

### Track 2. Dogfood A Real Capability

- `musu-computer-tools`를 실제로 완성하면서 회사 runtime을 도그푸딩
- helper-first Windows action
- split-host browser boundary
- browser/CDP consumer contract

### Track 3. Workforce Productization

- `Codex = manager`
- `BitNet = low-cost employee`
- queue assignment / supervisor / watchdog 운영 정책 강화

## 이 환경에서 가능한 것

- 코드 작성/수정
- 문서/계약/플랜 고정
- 실제 스크립트 실행과 proof 확보
- 회사 runtime 도그푸딩
- browser/helper/runtime validation

## 이 환경에서 없는 것

- 외부 사용자 피드백
- 결제/배포 채널
- 장기 production telemetry
- enterprise packaging

## 따라서 현재 MVP 정의

현재 MVP는 "AI 회사 운영체계로 실제 제품 capability 하나를 끝까지 완성하는 것"이다.

그 첫 대표 capability 후보는 `musu-computer-tools`다.
