# Market And Revenue Model

## 누가 고객인가

1. AI-native 개인 개발자
2. 소규모 제품 팀
3. 원격 운영 자동화가 필요한 팀
4. 보안/통제 요구가 큰 조직

## 돈은 어떻게 버는가

### 1. 구독형 SaaS

- seat + runtime + automation usage
- 팀 단위 workspace
- manager / worker / review lane

### 2. self-hosted / enterprise

- 온프레미스 배포
- approval / audit / governance 중심 판매

### 3. capability pack

- computer tools
- self-MCP
- remote realtime collaboration
- cross-device route/connectivity

## 왜 돈을 낼 것인가

유저는 단순 모델 접근이 아니라 아래 때문에 돈을 낸다.

- AI workforce 운영 가능성
- 반복 업무 자동화
- 로컬/원격 capability 통합
- review / approval / audit
- 팀과 프로젝트 단위 실행 체계

## 지금 환경에서 아직 없는 것

- 실제 고객 인터뷰
- 실사용 계약
- 결제 시스템
- 장기 운영 telemetry
- packaging / onboarding polish

## 지금 환경에서 할 수 있는 것

- 내부 도그푸딩
- 반복 운영 proof
- MVP capability 완성
- 가치가 분명한 한두 capability를 먼저 closing
