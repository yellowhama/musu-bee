## 🌟 Overview
CLI-Anything is a framework designed to make ANY software "Agent-Native" by wrapping it in a structured Command Line Interface (CLI). It bridges the gap between human-centric GUI software and AI agents by providing a deterministic, JSON-capable, and self-describing interface.

## 🏗️ Core Architectural Patterns

### 1. The 7-Phase Automation Pipeline
CLI-Anything uses a rigid methodology to generate CLIs:
1.  **Analyze**: Scans source/APIs to map capabilities.
2.  **Design**: Architects command groups and state models.
3.  **Implement**: Builds the Click-based Python CLI.
4.  **Plan Tests**: Creates E2E and Unit test plans.
5.  **Write Tests**: Implements the test suite.
6.  **Document**: Generates `SKILL.md` and user docs.
7.  **Publish**: Packages for PyPI/PATH.

**Application for MUSU**: We can use this pipeline to automate the creation of new `musu-computer-tools`.

### 2. The "Harness" Pattern (API > Scraping)
CLI-Anything avoids brittle pixel-clicking. It prioritizes:
- **Direct APIs**: Using REST (Ollama, Zoom) or Python SDKs (Blender `bpy`, iTerm2 API).
- **Accessibility Trees**: Using DOMShell (Browser) or UI Automation (Windows) to read the structural state of the UI.
- **Project Files**: Generating and rendering native project files (LibreOffice ODF, Inkscape SVG).

**Application for MUSU**: Our `ai-chat-spy` follows this by targeting Windows UI Automation (UIA) rather than raw screenshots.

### 3. The AI-Native Interface Trio
Every tool generated by CLI-Anything ships with:
- **REPL Mode**: For interactive human/agent exploration.
- **JSON Output**: Via a `--json` flag for deterministic agent parsing.
- **SKILL.md**: A self-contained "User Manual for AI" that enables autonomous tool discovery.

## 🛠️ Strategic Integration for `musu-indexer`

Based on this reference, we should enhance `musu-indexer` as follows:

1.  **Promote to "Harness Generator"**: Allow `musu-indexer` to not just index code, but also analyze APIs to help generate new `musu-functions`.
2.  **Standardize Tool Output**: Ensure every tool in `musu-computer-tools` supports a `--json` flag and returns structured data.
3.  **Dynamic Skill Discovery**: Implement a "Hub" logic where MUSU agents can query the local index to find and install new "computer tools" autonomously.
